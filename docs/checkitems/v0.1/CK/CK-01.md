# CK-01: Secure 속성 설정 여부

## 목적
HTTPS 환경에서 쿠키가 평문 전송되지 않도록 하기 위한  
Secure 속성 설정 여부를 확인한다.

## 점검 대상
- HTTP 응답의 `Set-Cookie` 헤더

## 점검 방법
- 대상 URL에 HTTPS 요청을 전송한다.
- 응답에 포함된 모든 `Set-Cookie` 항목을 수집한다.
- 각 쿠키에 `Secure` 속성 존재 여부를 확인한다.

## 판정 기준
- 확인됨(PASS): 모든 쿠키에 `Secure` 속성이 설정됨
- 설정 미흡(WEAK): 일부 쿠키에 `Secure` 속성이 설정되지 않음
- 미설정(MISSING): 모든 쿠키에 `Secure` 속성이 없음
- 확인불가(NOT_TESTED): `Set-Cookie` 헤더가 존재하지 않음

## OWASP Top 10 2025 매핑
- A07:2025 – Authentication Failures  
  (인증 상태 쿠키의 전송 보호 설정 미흡)

## 영향 가능성(Potential Impact)
- Secure 속성이 없는 경우,
  HTTPS 환경에서도 쿠키가 평문 전송될 가능성이 있다.

## 참고/전제(Conditions)
- HTTP 접근이 가능한 경우에만 실제 영향이 발생할 수 있다

## 증거(Evidence)
- 파싱된 `Set-Cookie` 항목 목록 (JSON)
