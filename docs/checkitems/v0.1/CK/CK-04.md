# CK-04: Cookie Path / Domain 명시 여부

## 목적
쿠키 적용 범위를 명확히 제한하여  
불필요한 인증 상태 노출을 방지하기 위한  
Path 및 Domain 속성 명시 여부를 확인한다.

## 점검 대상
- HTTP 응답의 `Set-Cookie` 헤더

## 점검 방법
- 대상 URL에 HTTP/HTTPS 요청을 전송한다.
- 응답에 포함된 모든 `Set-Cookie` 항목을 수집한다.
- 각 쿠키에 `Path` 및 `Domain` 속성 명시 여부를 확인한다.

## 판정 기준
- 확인됨(PASS): 모든 쿠키에 `Path` 또는 `Domain`이 명시됨
- 설정 미흡(WEAK): 일부 쿠키에만 `Path` 또는 `Domain`이 명시됨
- 미설정(MISSING): 모든 쿠키에 `Path` 및 `Domain`이 명시되지 않음
- 확인불가(NOT_TESTED): `Set-Cookie` 헤더가 존재하지 않음

## OWASP Top 10 2025 매핑
- A07:2025 – Authentication Failures  
  (인증 상태 쿠키의 적용 범위 관리 미흡)

## 영향 가능성(Potential Impact)
- 쿠키 적용 범위가 과도하게 넓은 경우,
  의도하지 않은 경로 또는 서브도메인에서
  인증 상태가 노출될 가능성이 있다.

## 참고/전제(Conditions)
- 다수의 애플리케이션 또는 서브도메인을 운영하는 경우

## 증거(Evidence)
- 파싱된 `Set-Cookie` 항목 목록 (JSON)
